## 一、新增模块：OCR PDF Reader

**文件**: `ocr_pdf_reader.py`（全新文件）

### 背景

原 `agno` 内置的 `PDFReader` 基于 pypdf 提取文本，对数学论文中的公式、特殊符号丢失严重。需要一套保留 LaTeX 公式的 OCR 方案。

### 实现

- 继承 `agno.knowledge.reader.pdf_reader.BasePDFReader`
- 使用 **PyMuPDF (fitz)** 将每页渲染为 PNG 图片（可配置 DPI）
- 图片 Base64 编码后发送至 **edusys OCR API**，返回含 LaTeX 公式的文本
- 支持 **多线程并发**（`ThreadPoolExecutor`，默认 4 线程），大幅加速多页 PDF 处理
- 支持 **结构化元数据注入**：`_create_documents_with_metadata()` 可为每页 Document 附加 section、element_types 等元数据

### 关键接口

| 属性/方法 | 说明 |
|---|---|
| `last_ocr_pages: list[str]` | `read()` 执行后，暴露每页原始 OCR 文本，供下游结构提取和原文存储使用 |
| `read(pdf, name, password, page_metadata)` | 主入口，返回 `List[Document]`，同时填充 `last_ocr_pages` |
| `_render_page_to_b64(page)` | 单页渲染为 Base64 PNG |
| `_ocr_single_page(page_index, b64_png)` | 单页 OCR，线程安全 |

### 配置示例（`PaperDive.py` 中）

```python
pdf_reader = OcrPDFReader(
    ocr_url="https://edusys5.sii.edu.cn/ocr",
    dpi=200,
    max_workers=4,
    chunking_strategy=semantic_chunking,
    split_on_pages=True,
)
```

## 二、新增模块：论文结构提取器

**文件**: `structure_extractor.py`（全新文件）

### 设计思路

采用**两阶段提取**（regex + LLM），兼顾速度与准确度：

1. **Phase 1 — 正则扫描**：用正则表达式从 OCR 文本中快速提取章节、定理、证明、定义、关键公式
2. **Phase 2 — LLM 精炼**：将正则结果 + 原文片段交给 LLM，修正误识别、补充遗漏、建立定理-证明关联、填写论文标题和概要

### 正则模式

| 模式 | 目标 | 示例匹配 |
|---|---|---|
| `_SECTION_RE` | 章节标题 | `1.2 Preliminaries` |
| `_THEOREM_RE` | 定理/引理/命题/推论 | `Theorem 3.1 ...` |
| `_PROOF_RE` | 证明起始 | `Proof.` / `证明` |
| `_DEFINITION_RE` | 定义 | `Definition 2.1 ...` |
| `_DISPLAY_MATH_RE` | 行间公式 | `$$...$$` / `\[...\]` |

### LLM 精炼输出结构

```json
{
  "title": "论文标题",
  "summary": "1-2 句概要",
  "sections": [{"id": "sec1", "title": "Introduction", "level": 1, "page": 1}],
  "theorems": [{"id": "thm3.1", "label": "Theorem 3.1", "type": "theorem", "statement": "...", "section_id": "sec3", "page": 5}],
  "proofs": [{"id": "prf_p5_120", "proves": "thm3.1", "page_start": 5, "page_end": 6}],
  "definitions": [{"id": "def2.1", "label": "Definition 2.1", "content": "...", "section_id": "sec2", "page": 3}],
  "key_equations": [{"id": "eq_p4_200", "latex": "...", "page": 4}]
}
```

### 新增：高层摘要提取 (`extract_paper_summary`)

在结构提取之外，新增 LLM 驱动的**论文高层摘要生成**，产出七维信息：

| 字段 | 类型 | 说明 |
|---|---|---|
| `title` | `str` | 论文标题 |
| `abstract` | `str` | 1-2 段中文概述 |
| `proof_approaches` | `dict` | 每个主要定理的证明思路（1-2 句） |
| `core_techniques` | `list[str]` | 核心方法/技巧列表 |
| `field_tags` | `list[str]` | 数学领域标签（2-3 个），如"图论""组合优化" |
| `content_tags` | `list[str]` | 研究内容关键词（2-4 个），描述"做了什么" |
| `technique_tags` | `list[str]` | 方法/技巧关键词（2-4 个），描述"怎么做的" |

三维标签（field / content / technique）设计目的是支持多维度快速筛选，避免仅按标题匹配的不准确问题。

### 辅助函数

| 函数 | 说明 |
|---|---|
| `find_section_for_page(structure, page_num)` | 根据结构查找某页所属章节 |
| `find_elements_on_page(structure, page_num)` | 查找某页包含的元素类型 |
| `format_structure_for_display(structure)` | 将结构 dict 格式化为可读 Markdown |

## 三、数据层：三张 SQLite 表

**文件**: `PaperDive.py`（`_init_tables()` 及配套 CRUD）

### 表结构

#### `paper_structures`（论文结构化数据）

```sql
CREATE TABLE IF NOT EXISTS paper_structures (
    paper_id TEXT PRIMARY KEY,
    structure_json TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)
```

#### `paper_pages`（每页 OCR 原文）

```sql
CREATE TABLE IF NOT EXISTS paper_pages (
    paper_id TEXT,
    page_num INTEGER,
    content TEXT,
    PRIMARY KEY (paper_id, page_num)
)
```

#### `paper_summaries`（LLM 摘要 + 三维标签）

```sql
CREATE TABLE IF NOT EXISTS paper_summaries (
    paper_id TEXT PRIMARY KEY,
    title TEXT,
    abstract TEXT,
    proof_approaches TEXT,   -- JSON
    core_techniques TEXT,    -- JSON
    field_tags TEXT,         -- JSON
    content_tags TEXT,       -- JSON
    technique_tags TEXT,     -- JSON
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)
```

### CRUD 函数

| 函数 | 操作 |
|---|---|
| `save_paper_structure(paper_id, structure)` | 写入/更新结构 |
| `load_paper_structure(paper_id)` | 读取结构 |
| `save_paper_pages(paper_id, pages)` | 批量写入每页原文 |
| `load_paper_pages(paper_id, start, end)` | 按页码范围读取原文 |
| `get_paper_page_count(paper_id)` | 获取论文总页数 |
| `save_paper_summary(paper_id, summary)` | 写入/更新摘要 |
| `load_paper_summary(paper_id)` | 读取完整摘要 |
| `load_all_paper_summaries()` | 读取所有论文的 paper_id + title + 三维标签（紧凑版） |

## 四、索引流程升级

**文件**: `PaperDive.py`（`_perform_scan()`、`load_paper_for_deep_analysis()`）

### 原流程

```
PDF → 语义分块 → 向量嵌入 → LanceDB
```

### 新流程（五步流水线）

```
PDF → OCR 读取
      ├─ 1. 存储 OCR 原文 → paper_pages 表
      ├─ 2. 结构提取（regex + LLM）→ paper_structures 表
      ├─ 3. 摘要生成（LLM）→ paper_summaries 表
      └─ 4. 语义分块 + 向量嵌入 → LanceDB（保留作为兜底检索）
```

### 容错设计

每个步骤独立包裹在 `try-except` 中：
- 原文存储失败 → 不影响结构提取和向量嵌入
- 结构提取失败 → 不影响摘要生成和向量嵌入
- 摘要生成失败 → 不影响向量嵌入
- 向量嵌入是基础步骤，失败则整篇论文索引失败

### 辅助函数

| 函数 | 说明 |
|---|---|
| `_extract_and_store_structure(paper_id, pages)` | 运行两阶段结构提取并存储 |
| `_extract_and_store_summary(paper_id, pages, structure)` | 运行 LLM 摘要提取并存储 |

## 五、新增工具：三级递进检索体系

**文件**: `PaperDive.py`

设计理念：从"纯向量 RAG"转变为"**结构化摘要 + 按需深读**"模型，Agent 先浏览高层摘要再决定是否深读，向量检索降为兜底手段。

### 检索层级

```
第一级：browse_paper_catalog()
        ↓ 从标题 + 三维标签筛出候选
第二级：get_paper_overview(paper_id)
        ↓ 看详细摘要、证明思路、章节大纲
第三级：read_paper_pages / read_paper_section
        ↓ 读 OCR 原文
兜底：  search_knowledge_base()（向量检索）
```

### 工具详情

#### `browse_paper_catalog()` — 全局索引

- 返回所有论文的紧凑目录：每篇含标题 + 三维标签（领域/内容/技巧）
- 输出示例：
  ```
  1. [sat-matching] Satisfiability and Matching in Boolean Algebra
     领域: 图论, 组合数学 | 内容: 匹配存在性, Hall定理推广 | 技巧: 构造性证明, 鸽巢原理
  ```

#### `get_paper_overview(paper_id)` — 详细概要

返回：
- 标题 + 三维标签
- 主要内容（abstract）
- 核心技巧列表
- 证明思路（按定理）
- 章节大纲（含页码）
- 定理/引理列表

#### `read_paper_pages(paper_id, start_page, end_page)` — 按页深读

- 从 `paper_pages` 表读取指定页码范围的 OCR 原文
- `end_page=0` 时只读一页
- 单次最多 10 页

#### `read_paper_section(paper_id, section_id)` — 按章节深读

- 根据 `paper_structures` 定位章节所在页码范围
- 自动读取该范围内的 OCR 原文
- 支持按 section_id（如 `sec2.1`）或章节标题匹配

## 六、新增工具：索引管理

**文件**: `PaperDive.py`

### `delete_paper_data(paper_id, targets)` — 选择性删除

支持按数据类型选择删除：

| targets 值 | 删除范围 |
|---|---|
| `vector` | 向量嵌入（LanceDB + knowledge_contents） |
| `structure` | 结构化数据（paper_structures 表） |
| `summary` | 摘要标签（paper_summaries 表） |
| `pages` | OCR 原文（paper_pages 表） |
| `all` | 以上全部（默认） |

支持逗号分隔多选，如 `targets="summary,pages"`。

### `reindex_paper(paper_id)` — 全量重索引

执行流程：
1. 删除该论文的全部旧数据（调用 `_delete_paper_data`）
2. 重新执行完整五步索引流水线
3. 返回结构提取和摘要标签的统计信息

## 八、Bug 修复

### Team Leader 工具调用失效

**现象**：Team Leader 尝试调用 `get_paper_structure` 时，输出为原始函数调用文本（如 `<function=get_paper_structure>...`）而非实际执行工具。

**根因**：在之前的 instructions 和 tools 列表更新过程中，`get_paper_structure` 被意外从 Team Leader 的 `tools` 列表中移除。由于 `agno` 框架的 agentic memory，LLM 仍"记得"该工具存在，但框架无法执行未注册的工具，导致输出原始文本。

**修复**：将 `get_paper_structure` 加回 `arxiv_team` 的 `tools` 列表。

```python
# 修复前
tools=[browse_paper_catalog, get_paper_overview, list_indexed_papers, save_note, list_notes]

# 修复后
tools=[browse_paper_catalog, get_paper_overview, get_paper_structure, list_indexed_papers, save_note, list_notes]
```
